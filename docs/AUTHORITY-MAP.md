## How to update this file

When an article publishes:

1. Find the article entry in the relevant cluster table.
2. Change `Status` from `Net New` or `Coming Soon` → `Published`.
3. Add the live URL to the entry (or to a new "URL" column if your row doesn't have one).
4. Update `Last updated` at the top.

When an article gets retired or merged:

1. Mark the old entry with `~~strikethrough~~` and a one-line note saying which article absorbed it.
2. Don't delete entries — the historical record matters.

When a new article is added that wasn't in the original 100:

1. Append a new row to the relevant cluster table.
2. Number it sequentially after the last entry in that cluster.
3. Update the cluster's article count in this file's headers.
