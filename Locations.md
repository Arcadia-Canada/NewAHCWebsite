# Arcadia Home Care — Location Pages SEO Strategy
**Last Updated:** March 2026
**Applies to:** All 10 active location pages

---

## THE #1 RULE

> Google will not rank location pages that look like duplicates.
> If you swap only the city name and nothing else, all pages compete
> against each other and none of them rank.
> Every page must have genuinely unique local content.

---

## WHAT MAKES A LOCATION PAGE RANK

Google ranks location pages that prove local relevance. That means:

1. **Local hospital references** — name the hospitals Arcadia works near
2. **Neighbourhoods served** — specific postal code areas, not just the city
3. **Local landmarks and transit** — TTC routes, GO stations, community centres
4. **Local demographics** — senior population, cultural communities
5. **Local health network context** — LHIN/Ontario Health region, CCAC history
6. **Unique introductory paragraph** — never copy-paste between pages
7. **LocalBusiness schema** — with areaServed matching the page

---

## PAGE STRUCTURE (use for every location page)

```
1. PageHero         — "[City] Home Care" — dark navy, city-specific headline
2. Intro paragraph  — unique to this city, mentions local context
3. Trust bar        — 5.0 stars, since 2005, clinical, 7 days
4. Services section — same 12 services (consistent is fine here)
5. Local context    — nearby hospitals, health network, neighbourhoods
6. Conditions cloud — same pill tags (consistent is fine here)
7. Testimonial      — ideally from a client in this region
8. How Care Starts  — 3-step summary
9. CTABlock         — phone + form
10. Related locations — links to nearby location pages
```

## WHAT NOT TO DO

- ❌ Never copy-paste the same paragraph across location pages with only the city name changed
- ❌ Never create a location page for a city you don't actively serve
- ❌ Never leave a location page with only a title and no content — Google sees thin pages as low quality
- ❌ Never use the same meta description across multiple location pages
- ❌ Never skip the LocalBusiness schema

---

## INTERNAL LINKING BETWEEN LOCATION PAGES

Every location page should link to:
- The parent region hub (e.g. North York → Toronto)
- 1–2 sibling location pages (e.g. North York → Scarborough)
- The dementia service page (highest priority service)
- The contact page

At the bottom of every location page add a "Also Serving Nearby:" section with links to adjacent location pages.
